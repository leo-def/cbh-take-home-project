# AI Agent Configuration for CBH Take-Home Project

> Configuration and capabilities for AI-assisted challenge solving.

## 🤖 Agent Capabilities

### Code Understanding
- **JavaScript ES6+**: Modern syntax, arrow functions, destructuring
- **Jest Testing**: Test structure, assertions, mocking
- **Refactoring Patterns**: Extract function, rename, simplify
- **Ticket Decomposition**: Breaking down features into tasks
- **Agile Estimation**: Story points and effort estimation

### Development Tasks Agent Can Perform

✅ **Can Help With**:
- Analyzing requirements and creating tickets
- Extracting reusable functions
- Renaming variables for clarity
- Simplifying complex logic
- Writing comprehensive Jest tests
- Identifying code smells and duplication
- Creating acceptance criteria
- Documenting ticket dependencies
- Performance optimization
- Error handling improvements

❌ **Not Recommended**:
- Changing fundamental algorithms without verification
- Breaking existing test cases
- Over-complicating simple logic
- Skipping test coverage for refactored code
- Creating unmaintainable or cryptic solutions

---

## 🛠 Recommended Skills

### Skill 1: Ticket Breakdown
**When**: Analyzing requirements document
**Process**:
1. Read requirements carefully
2. Identify distinct work items
3. Create ticket template with title, description, criteria
4. Estimate effort for each ticket
5. Map dependencies (e.g., Ticket 2 depends on Ticket 1)
6. Organize into logical sequence

### Skill 2: Code Refactoring
**When**: Improving code quality
**Process**:
1. Understand current code behavior
2. Identify code smells (duplication, long functions)
3. Extract functions with single responsibility
4. Rename for clarity
5. Simplify conditionals and loops
6. Add tests for refactored code

### Skill 3: Test-Driven Refactoring
**When**: Refactoring with confidence
**Process**:
1. Verify all original tests pass
2. Add tests for edge cases
3. Refactor code incrementally
4. Verify tests still pass
5. Repeat until satisfied with quality

---

## 🔑 Key Concepts

- **Code Smell**: Indicator of deeper problem (duplication, complexity)
- **Single Responsibility**: Function should do one thing well
- **DRY Principle**: Don't Repeat Yourself - extract common logic
- **Test Coverage**: Verify refactoring doesn't break functionality
- **Ticket Estimation**: Realistic effort assessment
