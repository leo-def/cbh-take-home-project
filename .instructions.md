# Development Instructions for AI Agents - CBH Take-Home Project

> Guidelines for AI-assisted development of coding challenge solutions.

## 🎯 Project Overview

- **Purpose**: Solve two coding challenges (Ticket Breakdown + Refactoring)
- **Type**: Coding Challenge Suite
- **Tech Stack**: JavaScript (Node.js 16+), Jest for testing
- **Build**: npm scripts

---

## 🏗 Architecture Pattern

**Two Independent Challenges**:

**Challenge 1 - Ticket Breakdown**:
- Analyze requirements document
- Decompose into logical implementation tickets
- Estimate effort and define dependencies
- Document in Markdown with clear structure

**Challenge 2 - Refactoring**:
- Improve existing code quality
- Extract reusable functions
- Reduce duplication and complexity
- Maintain functionality with improved tests

---

## 📂 File Naming & Conventions

**Functions**: camelCase with descriptive names
- `calculateTax()`, `applyDiscount()`, `validateInput()`

**Variables**: camelCase
- `basePrice`, `discountAmount`, `isValid`

**Constants**: UPPER_SNAKE_CASE
- `MAX_DISCOUNT`, `MIN_QUANTITY`, `TAX_RATE`

**Test Files**: descriptive with `.test.js` suffix
- `ticketBreakdown.test.js`, `refactoring.test.js`

**Test Functions**: `describe()` and `test()` blocks
- Clear, readable test names

---

## ✅ DO's

1. **DO** write comprehensive tests alongside refactored code
2. **DO** extract functions with single responsibility
3. **DO** use descriptive variable and function names
4. **DO** document ticket dependencies and estimates
5. **DO** validate input parameters with error handling

---

## ❌ DON'Ts

1. **DON'T** break existing functionality when refactoring
2. **DON'T** create deeply nested or complex logic
3. **DON'T** skip test coverage for refactored code
4. **DON'T** hardcode magic numbers (use constants)
5. **DON'T** create tickets that are too vague or too granular

---

## 📋 Common Workflows

**Ticket Breakdown Process**:
1. Read requirements document carefully
2. Identify distinct features/components
3. Break into 5-10 logical tickets
4. Add realistic estimates (2-5 hours each)
5. Map dependencies between tickets
6. Document acceptance criteria
7. Review for completeness

**Refactoring Process**:
1. Understand existing code behavior
2. Identify code smells (duplication, complexity)
3. Extract reusable functions
4. Rename for clarity
5. Simplify conditional logic
6. Add/improve tests
7. Verify functionality unchanged

**Running Tests**:
```bash
npm install          # Install dependencies (jest)
npm test             # Run test suite
npm test -- --coverage  # Generate coverage report
npm start            # Run challenge application
```
